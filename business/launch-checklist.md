# Galeria Launch Checklist

> Everything needed before accepting first paying customer
> Last updated: 2025-03-04

---

## Revenue-Ready (Critical)

### Payments

- [ ] Create Stripe account
- [ ] Add Stripe integration to app
- [ ] Test checkout flow end-to-end
- [ ] Set up webhooks for payment events
- [ ] Configure tax calculation (Stripe Tax)
- [ ] Test refund process
- [ ] Add upgrade prompts at limit boundaries
- [ ] Create "downgrade" flow (if needed)

### Pricing Display

- [ ] Pricing page reflects correct amounts
- [ ] Feature comparison is clear
- [ ] "Upgrade" CTAs are prominent
- [ ] FAQ covers billing questions
- [ ] Terms of payment are clear

---

## Legal & Compliance

### Documents (Publish these)

- [ ] Terms of Service
- [ ] Privacy Policy
- [ ] Refund Policy
- [ ] Cookie Policy (if using tracking)
- [ ] Data Processing Agreement (for EU customers)

### Business Entity

- [ ] Form LLC (recommended) or decide on structure
- [ ] Get EIN (if in US)
- [ ] Open business bank account
- [ ] Understand tax obligations

### Compliance

- [ ] GDPR compliance (if targeting EU)
- [ ] CCPA compliance (if targeting California)
- [ ] Data breach response plan
- [ ] Security review (password hashing, encryption)

## Pre-Launch Testing

### Beta Testing

- [ ] 5-10 beta users from target niche
- [ ] Collect structured feedback
- [ ] Run 2-3 test events
- [ ] Fix critical bugs

### Payment Testing

- [ ] Test $1 transaction
- [ ] Test subscription creation
- [ ] Test upgrade flow
- [ ] Test cancellation flow
- [ ] Verify webhook handling

### Load Testing

- [ ] Test with 100 concurrent users
- [ ] Test photo upload with large files
- [ ] Test database under load

- [ ] All critical bugs fixed
